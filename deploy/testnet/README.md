# OctraVPN public testnet — operator bring-up

This directory holds everything an operator needs to stand up the
OctraVPN **public testnet** tier. Testnet sits between devnet (100%
local, ephemeral, no DNS — see `docker/devnet/`) and mainnet (real
OCT, ceremony-signed — see `ceremony/`):

| Tier     | Where               | Public? | Persistent? | Permissioned? |
| -------- | ------------------- | ------- | ----------- | ------------- |
| Devnet   | `docker/devnet/`    | no      | no          | no            |
| Testnet  | **this dir**        | yes     | yes         | yes (operator multisig) |
| Mainnet  | `ceremony/`         | yes     | yes         | yes (3-of-5 multisig)   |

## What's in the box

A 3-validator + 1-mesh-control + 1-DERP + Prometheus + Grafana stack,
brought up by one Docker Compose file (`docker-compose.testnet.yml`
at repo root). Validators run the **same binary** the devnet harness
builds (reusing `docker/Dockerfile.node`); only the runtime config
and the surrounding infrastructure differ.

### Validator capabilities (default)

| Node       | signer | observer | relay | bridge |
| ---------- | ------ | -------- | ----- | ------ |
| validator1 |   ✓    |    ✓     |       |        |
| validator2 |   ✓    |    ✓     |       |        |
| validator3 |   ✓    |    ✓     |   ✓   |        |

`bridge` is not enabled on testnet v0.1 (no cross-chain plumbing).
`relay` puts validator3 in the DERP fallback path for clients behind
symmetric NAT — exactly one relay-capable validator is enough.

## Prerequisites

1. **Docker Engine ≥ 24.0** with the Compose v2 plugin
   (`docker compose version` should print 2.x).
2. **A funded set of wallets** generated by
   `scripts/testnet/genesis-wallets.sh` — see "Bootstrap sequence"
   below. Each validator needs ≥ `MIN_ENDPOINT_STAKE` OCT; the
   operator multisig needs enough for tx fees + any program redeploy
   you do.
3. **Public DNS records** for every endpoint advertised in
   `.env.testnet`:
   - `tn1.example.com` → validator1 host (A/AAAA)
   - `tn2.example.com` → validator2 host
   - `tn3.example.com` → validator3 host
   - `mesh.testnet.example.com` → host running mesh-control
   - `derp.testnet.example.com` → host running DERP
4. **Firewall rules** opening:
   - WireGuard UDP on every validator's public port (51820, 51830,
     51840 by default)
   - TCP 51821–51823 (control-plane / Prometheus scrape)
   - TCP 8085 to mesh-control (operator API)
   - TCP 3340 + UDP 3478 to DERP
   - TCP 3000 (Grafana) **bound to operator VPN / loopback only** —
     do not expose Grafana to the public internet
   - TCP 9090 (Prometheus) — same constraint

## Bootstrap sequence

The script driving this is `scripts/testnet/bootstrap.sh`. It runs
steps 4–10 below in order; steps 1–3 are operator-driven prerequisites.

### Step 1 — generate genesis wallets

```sh
export OCTRAVPN_KEY_PASSPHRASE='use a real value, not this string'
./scripts/testnet/genesis-wallets.sh \
    --out  ./deploy/testnet/state/genesis \
    --threshold 2 --signers 3
```

This writes sealed bundles for the three validators and the 2-of-3
operator multisig under `state/genesis/`. It also writes a
`MANIFEST.txt` recording the sha256 of every public key — keep this
manifest somewhere out-of-band (you'll need it to verify the on-chain
state matches your genesis after the first bond round).

### Step 2 — fund every wallet

Take every `wallet.pub` and `multisig.addr` from
`state/genesis/MANIFEST.txt` and request faucet drops from
[the Octra faucet](https://faucet.octra.network/). Wait for chain
confirmation (≈ 14 s per epoch on devnet) before proceeding.

### Step 3 — fill in `.env.testnet`

```sh
cp deploy/testnet/.env.testnet.example deploy/testnet/.env.testnet
$EDITOR deploy/testnet/.env.testnet
```

Replace every `PLACEHOLDER_*` value with the real one. In particular:

- `VALIDATOR{1,2,3}_ADDR` — read from
  `state/genesis/validator{1,2,3}/wallet.pub`
- `VALIDATOR{1,2,3}_PUBLIC_ENDPOINT` — DNS:port from step 0
- `OPERATOR_MULTISIG_ADDR` — `cat state/genesis/multisig/multisig.addr`
- `OPERATOR_MULTISIG_THRESHOLD` — 2 (matches the genesis script's default)
- `PROGRAM_ADDR` — paste once you've deployed the v3 program (next step)
- `OCTRAVPN_KEY_PASSPHRASE` — same value you used in step 1
- `GRAFANA_ADMIN_PASSWORD` — anything strong; default admin/admin is rejected

`bootstrap.sh` refuses to proceed if any `PLACEHOLDER_` remains.

### Step 4 — deploy the program

Outside the scope of bootstrap.sh — use `octra forge create` against
the chosen RPC, with the operator multisig as the eventual owner.
See `docker/devnet/README.md` §"Deploy `program/main.aml`" for the
exact invocation. Copy the deployed `address` field into
`PROGRAM_ADDR` in both `.env.testnet` and
`testnet-params.toml`.

### Step 5 — bootstrap

```sh
./scripts/testnet/bootstrap.sh deploy/testnet/testnet-params.toml
```

This runs:

1. Parse `testnet-params.toml`. Refuses any tier other than
   `testnet`. Refuses a mainnet-looking RPC URL unless
   `--allow-mainnet` is passed.
2. Validate `.env.testnet` has no PLACEHOLDERs.
3. Render per-validator + mesh-control configs from
   `templates/` into `state/<node>/`.
4. Render `state/prometheus/prometheus.yml` and `targets.json`.
5. `docker compose config -q` sanity check.
6. Build / pull images.
7. Start **mesh-control** first; wait for `/v1/health`.
8. Start validator1, validator2, validator3.
9. Start DERP.
10. Start Prometheus + Grafana.

Add `--dry-run` to walk every step without touching docker.

### Step 6 — verify

```sh
./scripts/testnet/health-check.sh
```

Six independent checks, each pass/fail. Non-zero exit if any failed.

## Peer discovery — how a new operator joins

The testnet is **permissioned-public**: anyone can reach the
endpoints, but only operators on the roster can publish receipts that
will be honoured by the rest of the mesh. New operators get on the
roster by redeeming a **preauth key**.

### Existing operator: issue a preauth key

```sh
# This call must be signed by ≥ threshold members of the operator
# multisig. For testnet, that's any 2 of the 3 multisig signers.
curl -X POST https://mesh.testnet.example.com/v1/preauth \
     -H "Authorization: Bearer $MULTISIG_AUTH" \
     -d '{"role": "signer,observer", "ttl_secs": 86400}'
# -> {"preauth_key": "ABCD...", "expires_at": "..."}
```

Give the new operator the `preauth_key`. It's valid for 24 h (set by
`preauth_ttl_secs` in `templates/mesh-control.toml.template`).

### New operator: redeem it

```sh
./scripts/testnet/join.sh \
    --preauth-key ABCD...32CHARS \
    --control-url https://mesh.testnet.example.com \
    --endpoint     tn4.example.com:51820
```

On success: prints `JOINED <pubkey>` and writes `./operator-state/`
containing the new sealed wallet key + a signed roster envelope.
On failure: prints `JOIN_ERROR: <code> <message>` (see `join.sh`
header for the code list).

## Revocation — evict a misbehaving peer

Revocation is a multisig action: any `OPERATOR_MULTISIG_THRESHOLD`
signers can sign a `revoke` envelope and POST it to mesh-control:

```sh
curl -X POST https://mesh.testnet.example.com/v1/revoke \
     -H "Authorization: Bearer $MULTISIG_AUTH" \
     -d '{"pubkey": "<the-peer-pubkey>", "reason": "double-sign-evidence-hash"}'
```

mesh-control verifies the threshold of signatures, appends to the
revocation log (persisted on the `mesh-control-state` volume), and
pushes a new roster to every validator. Validators drop sessions
attached to the revoked pubkey at the next epoch boundary.

For chain-side equivocation slashing, see
`docs/testnet.md` §"Validator-equivocation slashing" — that's a
separate flow (submit double-sign evidence to the program) and is
the **economic** punishment; revocation is the **operational** one
and they're complementary, not redundant.

## Persistent volume layout

The compose file declares 13 named volumes. They split into three
groups:

**Per-validator (3 × 3 = 9 volumes)**

| Volume                  | Maps to inside container       | Contents                                              |
| ----------------------- | ------------------------------ | ----------------------------------------------------- |
| `validator<N>-state`    | `/var/lib/octravpn`            | chain head cache, peer-roster snapshot, P1-8/P1-9 receipt journal, settle ledger |
| `validator<N>-keys`     | `/etc/octravpn/keys` (ro)      | sealed wallet bundle, sealed WG noise key             |
| `validator<N>-audit`    | `/var/log/octravpn-audit`      | append-only audit log mirror — ship to a SIEM         |

**Mesh control + DERP**

| Volume                | Maps to inside container       | Contents                                              |
| --------------------- | ------------------------------ | ----------------------------------------------------- |
| `mesh-control-state`  | `/var/lib/octravpn`            | sqlite of preauth keys + redeemed joins + revocation log (CRITICAL — back up out-of-band) |
| `mesh-control-keys`   | `/etc/octravpn/keys` (ro)      | sealed mesh-control signing key (separate from validators) |
| `derp-state`          | `/var/lib/derper`              | STUN endpoint cache (regeneratable)                   |

**Observability**

| Volume              | Maps to inside container | Contents                                  |
| ------------------- | ------------------------ | ----------------------------------------- |
| `prometheus-data`   | `/prometheus`            | TSDB (retention = `PROMETHEUS_RETENTION`) |
| `grafana-data`      | `/var/lib/grafana`       | dashboards, users, alert state            |

Plus the operator-mounted bind dir at `$TESTNET_CONFIG_DIR` (default
`./deploy/testnet/state/`) which is **not** a named volume — it
holds rendered TOML configs + sealed key files + the prometheus
targets json. Bootstrap writes here; the containers mount it
read-only.

## Operator decision points (parameterized in `testnet-params.toml`)

These are the knobs an operator must consider; defaults are sane for
a first bring-up.

| Param                       | Default               | What's the trade-off?                              |
| --------------------------- | --------------------- | -------------------------------------------------- |
| `tier`                      | `testnet`             | Must be `testnet`; guard rail against running this on mainnet |
| `network_name`              | `octra-testnet-v0`    | Cosmetic — appears on dashboards + join URL banner |
| `rpc_url`                   | `devnet.octrascan.io` | Real Octra testnet RPC when it exists              |
| `chain_id`                  | `octra-testnet`       | Pinned into receipt signing domain (P1-5)          |
| `program_addr`              | (empty — must be set) | Address of deployed v3 program                     |
| `validators[].endpoint`     | tn{1,2,3}.example.com | Real DNS — DO NOT use loopback                     |
| `validators[].roles`        | varies                | Default: v1/v2 signer+observer, v3 +relay          |
| `validators[].price_per_mb` | 100                   | Operator pricing; equal on testnet to avoid arb    |
| `mesh_control_hostname`     | mesh.testnet.example  | Real DNS                                           |
| `multisig_threshold`        | 2                     | Tighter than mainnet (3); accept easier rotation   |
| `multisig_signer_count`     | 3                     | Same logic                                         |
| `derp_hostname`             | derp.testnet.example  | Real DNS — DERP must be reachable from the open net|
| `prometheus_retention`      | 30d                   | Storage cost vs. forensics window                  |
| `config_dir`                | ./deploy/testnet/state| Bind-mount root for rendered configs               |
| `genesis_dir`               | $config_dir/genesis   | Where genesis-wallets.sh writes sealed bundles     |

## Tear-down

```sh
# Stop everything, keep volumes (state survives):
docker compose --env-file deploy/testnet/.env.testnet \
    -f docker-compose.testnet.yml --profile testnet down

# Stop AND wipe volumes — DESTRUCTIVE; chain history is gone:
docker compose --env-file deploy/testnet/.env.testnet \
    -f docker-compose.testnet.yml --profile testnet down -v
```

`mesh-control-state` is the most precious volume — it contains the
roster + revocation log. Lose it and you must run a fresh ceremony
(every operator re-joins with new preauth keys).

## Troubleshooting

| Symptom                                       | Likely cause                                        | Fix                                                                 |
| --------------------------------------------- | --------------------------------------------------- | ------------------------------------------------------------------- |
| `bootstrap.sh` fails at step 2                | PLACEHOLDER left in `.env.testnet`                  | grep `grep PLACEHOLDER deploy/testnet/.env.testnet`                 |
| mesh-control never becomes healthy            | `OPERATOR_MULTISIG_ADDR` empty / not chain-bonded   | re-run genesis-wallets.sh, fund the multisig, restart               |
| `health-check.sh` step 2 fails                | RPC URL unreachable from host                       | `curl $OCTRA_RPC_URL` directly                                      |
| `health-check.sh` step 3 fails                | validator container crashed                         | `docker compose logs validator1` — usually a config-render issue    |
| `health-check.sh` step 4 = 0 receipts         | no settlement traffic yet                           | run a client session against one of the validators                  |
| `join.sh` returns `E_PREAUTH_INVALID`         | preauth expired or already redeemed                 | issue a fresh one via `/v1/preauth`                                 |
| Grafana shows no data                         | targets.json not picked up                          | restart prometheus; check `state/prometheus/targets.json` exists    |

See `docs/testnet.md` for the full list of features that work
against a real testnet vs. still require the in-process mock.
